Python and GIS

Why choose python?

I chose python due to Due to its simplicity, it is simple to learn and comprehend, making it an excellent choice for beginners. Python can be used for several tasks because it is a flexible language, including web development, game development, machine learning, and artificial intelligence. It is what we would describe as a high level programming language, which means that it is typically less complicated and a bit more user friendly than more low level languages such as C, C++ or Java. The general rule is that high level languages make it quicker and easier to write code, but at the expense of those programs being slightly slower when they execute. The exciting thing about Python is that it can link up with lower-level languages such as C or C++ through the use of libraries, which means that you can get the best of both worlds: code that is simple and efficient to write, but which runs quickly too. This means it handles complex and otherwise mundane tasks for us such as reading and writing GIS data files, rendering maps and transforming coordinates, for example.

Moving Forward into GIS

As I looked to move away from simple data and start using geospatial data, I began to incorporate everything I had learnt into GeoPython projects. Upon reflection, the best order to have done this in would have included:

  1. Washington DC Construction Permits Project: This is a great introduction to spatial data, but the data manipulation remains largely focused on the core data analysis techniques learnt above, and focuses on spatial analysis for producing an output and using a sjoin. As such it would have been a great place to start.
  2. Bogota Analysis: This is a perfect introduction to data and analysis with an exclusively spatial outlook and goal.
  3. Shortest Border Algorithm: This project has an amazing introduction to geographic data issues and core themes, such as measurign distance across ellipsoidal representations of the earth.
  4. Floodfill Model: This has a great cross over with catastrophe analytics and models, as this in many ways simulates one stochastic event and its outcome.
  5. Agent Based Modelling: This is a great final project as it allowed me to look further into simulations, focusing more on the human element than the natural catastrophe element. This draws parallels in certain applications of these models to evacuation response and simulations of human movement in response to a stimuli
